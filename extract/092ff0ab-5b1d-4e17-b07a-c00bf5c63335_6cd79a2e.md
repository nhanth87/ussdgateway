# Service Discrimination

## Overview

## I ntroduction

When a message for a new TCAP dialog is received by the SLEE TCAP interface, it determines which SLEE application and service the dialog should be set to for processing. This decision is made by the SLEE based on either:

- the SLEE service key as determined by the TCAP Interface and the configuration defined in SLEE.cfg, or

- through the use of a SLEE Correlation ID pre-allocated by the SLEE and determined by the TCAP Interface.

## Service Keys

## I ntroduction

The TCAP Interface can use a SLEE Service Key to initiate a dialog to a SLEE service.

The SLEE Service Key used by the TCAP Interface is made up from the following elements:

1 Base service key value defined by interface configuration parameter

2 Destination SCCP subsystem number (SSN)

Integer number retrieved from the message in the conveyed protocol. The value derived is dependent on the protocol:

## Correlation IDs

## I ntroduction

The TCAP Interface can also use a SLEE Correlation ID to initiate a dialog to a SLEE service. The correlation ID is allocated by a SLEE application running a service and sent out from the SLEE to an entity that will initiate a TCAP dialog back to the SLEE.

![](extract/subset_41_45_a077339e_b54bfe06/images/page_3_img_in_image_box_165_389_214_429.jpg)

When the TCAP Interface receives the TC-BEGIN of this new TCAP dialog, it can initiate a new SLEE dialog to the same application by using the correlation ID. The correlation ID must be read from within the TCAP component holding the conveyed protocol's first message.

## Conveyed Protocol Support

## I ntroduction

This topic details the behaviour of the TCAP Interface when dealing with supported conveyed protocols as defined by the proto configuration parameter.

If the protocol is not supported, the TCAP Interface attempts to initiate a new SLEE dialog based on a default service key as defined in CS-1 INAP.

If the proto configuration parameter is set to inap, the TCAP Interface will support INAP operations.

The TCAP Interface examines the first conveyed component and expects one of the following INAP operations:

InitialDP

• AssistRequestInstructions

## initialDP

If the operation is an initialDP, the TCAP Interface attempts to initiate a SLEE dialog with a service key using the initialDP's serviceKey parameter for the conveyed protocol's constituent.

Here are some example lines suitable for inclusion in SLEE.cfg.

- If the INAP SSN = 13 (0x0d) and the TCAP Interface is using the default base key of 0x1, this line would route all IDPs with service key 8 to the SLEE service Serv1:

SERVICEKEY=INTEGER 0x10d0000008 Serv1

- If the INAP SSN = 112 (0x70) and the TCAP Interface is using a non-default base key of 0x1234, this line would route all IDPs with service key 16 to the SLEE service Serv2.

SERVICEKEY=INTEGER 0x12347000000010 Serv2

## assistRequestInst ructions

If the operation is an assistRequestInstructions, the TCAP Interface attempts to initiate a SLEE dialog using a correlation ID. The correlation ID used is a decimal conversion of the digits from the assistRequestInstructions' correlationID parameter.
