## I ntroduction to SLEE TCAP Interfaces

## I ntroduction

A SLEE TCAP Interface is a SLEE interface that enables the SLEE to inter-work with a TCAP Protocol stack.

The interface converts messages arriving from the TCAP Protocol stack and converts them into SLEE Events. The SLEE Events are then sent on to the application which is configured to service the call. The TCAP Protocol stacks likewise convert SLEE Events coming from a SLEE Application back into a form the TCAP stack can understand.

A TCAP protocol stack is a software implementation of a networking protocol suite where a group of protocols (in this case TCAP and the applicable underlying protocols such as SCCP/MTP or SUA/SCTP/IP) all work together to allow software and hardware (i.e. via a set of NIC or SS7 Interface Cards on an applicable high-performance server-based platform) to communicate with a telecommunications network. The networks typically support Intelligent Network-type Signalling Control and other associated or similar functions.

## TCAP Interface layers

The Oracle TCAP Protocol Stack used in the specific installation can vary depending on the requirements of the network and the type of physical interface and network protocols used. Generally the final TCAP protocol stack can be built from multiple layers of smaller protocol stacks that sit below the TCAP layer of the SS7 System. These underlying layers or smaller protocol stacks may be comprised of traditional SS7 protocols (SCCP/MTP) or more recent TCP/IP or Sigtran protocols (SUA/SCTP/TP) and may be provided and supported by Oracle and/or third party vendors such as Hughes or Stratus (SINAP). The point of interface between the SLEE TCAP Interface and the underlying third-party Protocol Stack is an important boundary as it determines a discrimination point for Oracle and non-Oracle software. At this point there is typically a distinct change:

- in software architecture, style, technology and hence configuration/installation and operational procedures.

- the type of low-level diagnostics tools available.

- the actual protocol (or, from a software development viewpoint, the API) being used at the interface point. The 2 protocols typically used at the interface point are:

SCCP or

☑ TCAP.

An SCCP Level interface would use an Oracle level TCAP implementation and would be more flexible and consistent with other Oracle SLEE TCAP interface variants.

A TCAP Level interface would use the specific third-party TCAP level implementation supplied.

A SLEE Event is a unit or packet of data that processes connected to the SLEE use to communicate with each other.

## SCCP and TCAP level interfaces

The SLEE TCAP Interface is layered to allow for easy porting to various TCAP and SCCP Protocol stacks.

Interfaces which use TCAP level Protocol stacks include SINAP TCAP stack.

Interfaces which use SCCP level Protocol stacks include:

• Hughes SCCP Stack, and

• SINAP SCCP Stack.

The Oracle TCAP stack contains a logging capability that can be used to store messages received and sent by the stack to SCCP.

A SCCP-level TCAP interface can be toggled into and out of logging mode by sending the HUP signal to the running interface.

Once logging is switched on the interface will append to a binary log file in /tmp. Output is in the form of raw data which can be decoded with the tcread programme.
