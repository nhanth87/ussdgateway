The TCP TCAP interfaces communicate with each other over standard TCP/IP connections and can be configured to run over dual (redundant) networks with each node accessible via two independent network interfaces (i.e. with separate addresses).

Each TCP TCAP node is uniquely identified by an integer value equivalent to a traditional SS7 network node's point-code and can communicate with other nodes on the same SSN (communication between different SSNs is not possible).

![](extract/subset_36_40_1f776152_342327c7/images/page_1_img_in_image_box_296_1069_678_1213.jpg)

The mapping of the node's point-code to its network IP address(es) is performed by standard host lookup (e.g. via DNS or from the /etc/hosts file).

The node’s primary (or only) address must be assigned the name:

- pc- $ ^{<} $ pointcode>, where pointcode is the node’s point-code value.

The node’s secondary (which is optional) address can either be associated as a second address to the same name or assigned the name:

• pc-<pointcode>-2, where pointcode is the node’s point-code value.

## X ML/TCAP Interface Configuration

The xmlTcapInterface receives BPL requests from the smsTrigDaemon or from a third party. Each request is used to construct an InitialDP which is sent to ACS via the SLEE. The xmlTcapInterface returns an indication of whether or not the request was successful through an HTTP response.

The InitialDP contains the following information:

• calling party number (from the xml request),

• called party number (from the xml request),

• PIN digits.

• Extension fields (from the xml request), and

• Service key for the application required.

eserv.config xmlTcapInterface supports the following parameters in the CCS section of configuration eserv.config.

xmlTcapInterface = {
    listenPort = 3072
    idp_sk = 10
}

## Parameters

listenPort

Defines the listening port for incoming requests.

Default: 3072

idp sk

The IDP service key for the application to insert into send the InitialDP.

Default: 10

## Start up

The following entries must be included in the SLEE.cfg configuration file for the xmlTcapInterface to work:

SERVICE=CCS_BPL 1 slee_acs CCS_BPL
SERVICEKEY=INTEGER 50 CCS_BPL

INTERFACE=xmlTcapInterface xmlTcapInterfaceStartup.sh /IN/service_packages/SLEE/bin/ UDG

Note: For further information on configuring the SLEE, see the SLEE Technical Guide.
