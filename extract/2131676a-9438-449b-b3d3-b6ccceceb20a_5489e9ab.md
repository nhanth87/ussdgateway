## Configuration Overview

## I ntroduction

This topic provides a high level overview of how the SLEE TCAP Interfaces are configured.

Both the TCAP and SCCP level interfaces use inheritable TCAP and SCCP classes to provide consistent configuration between all SLEE TCAP interfaces of the same level.

The TCAP interfaces are configured using the following:

• options set in the startup shell script, and/or

• options set in the common configuration file, tcapif.def.

All interfaces have:

• generic parameters that apply to all types of that level interface, and

- specific parameters f or the 3rd party SS7 or IP stack that the interface is using.

This table describes the steps involved in configuring the TCAP IF for the first time.

1 The environment the TCAP stack will run in must be configured correctly. This includes:

- setting the location of the SLEE base directory (if this is different from default), and

• configuring the location of the SCCP log file.

For more information about configuring environmental variables, see Environmental variables (on page 14).

2 If required, changing the default startup variables in the SLEE.cfg file.

3 Any non-default command line parameters must be added to the relevant TCAP startup shell script(s):

• hssSclf.sh (Hughes)

• sinSclf.sh (SINAP)

• sinTclf.sh (SINAP), or

tcpTclf.sh (TCAP).

For the full list of parameters, see the relevant Interface Configuration sections.

4 Protocol specific configuration data must be configured for the specific stack. For example, for Hughes: the normal peers and signalling gateways in the network must be added to peers.conf. For more information, see Peers file configuration.
